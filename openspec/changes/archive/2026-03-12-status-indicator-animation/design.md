## Context

The current pane status indicators are static Unicode symbols rendered inline with each pane item. The existing 100ms tick mechanism already refreshes the UI, providing a natural hook for animation. Status indicators are generated by `PaneStatus.Indicator()` in the monitor package.

## Goals / Non-Goals

**Goals:**
- Animated spinner for Running status that cycles every 100ms
- Green colored circle for Ready status
- Maintain current Idle indicator (○)

**Non-Goals:**
- Custom animation speeds (we use the existing 100ms tick)
- User-configurable indicator styles
- Animation for other states

## Decisions

### Decision 1: Use braille spinner pattern ⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏

**Choice:** Use braille dot spinner characters that cycle through 10 frames.

**Rationale:**
- Compact single character width
- Smooth rotation appearance
- Well-supported in modern terminals
- Alternative: Use simpler |/-\ but looks less polished

### Decision 2: Pass frame counter to Indicator method

**Choice:** Add `IndicatorAnimated(frame int)` method that accepts a frame counter.

**Rationale:**
- Keeps animation state in UI layer (where ticks happen)
- Monitor package stays stateless for status
- Alternative: Store frame in monitor - but monitor shouldn't own UI timing

### Decision 3: Use lipgloss for green color

**Choice:** Apply lipgloss foreground color to Ready indicator in the UI layer.

**Rationale:**
- Consistent with existing styling approach in app.go
- Keeps indicator generation simple (just returns character)
- Color application happens at render time

## Risks / Trade-offs

**[Trade-off] Animation adds visual noise** → Acceptable because it provides valuable feedback about pane activity. Users expect busy indicators to move.

**[Risk] Terminal doesn't support braille** → Fallback would show garbled text. Mitigation: Braille has wide support in modern terminals; this is an acceptable limitation for a tmux-focused tool.
